## 1. 目标

v0.1 把 BioSpark-Light 做成了能双击运行的 .exe，但任何拿到 zip 的人都能无限使用，没有任何访问控制。v0.2 给桌面版加上**分发门槛 + 试用 → 付费激活**的两层体系，目标按优先级：

1. **社区建设** — 收集用户姓名 / 邮箱 / 机构，便于跟进反馈
2. **防止无限扩散** — 不公开下载链接，下载需经审批
3. **免费试用 → 付费解锁** — 5 次免费训练，超限弹付款墙

联网要求：只在首次激活时联网（甚至可以离线），激活后完全离线运行。

成功判定（DoD）：

1. 全新安装 → 横幅显示 "试用模式 · 还剩 5 次"
2. 训练 5 次后 → 第 6 次训练请求返回 HTTP 402 → 前端弹付款墙
3. 输入合法激活码 → 横幅切换为 "已授权 · email · 至 YYYY-MM-DD" → 后续训练不再被拦截
4. 篡改激活码 / 已过期激活码 → 后端拒绝，前端显示原因
5. 数据整理页文案修正 — `.csv` 改为 `.csv/.txt`，对 OpenBCI 用户友好

全部达成。

## 2. 关键决策与理由

### 2.1 双层架构：分发门槛 + 应用内许可

| 层 | 谁来执行 | 改动量 |
|----|---------|--------|
| 第一层：GitHub Release 不公开 | 开发者手工管理（仓库 unlisted、下载链接通过表单审批） | 0 行代码 |
| 第二层：应用内试用计数 + 激活码 | 软件本身的逻辑（这层是本次 PR 的内容） | ~400 行 |

第一层零代码，避免过度工程；第二层确保即使第一层泄露（用户把 zip 转给朋友），新副本仍受试用次数限制。

### 2.2 离线 HMAC-SHA256，不引服务端

激活码格式：

```
BSL2-{base64url(json_payload)}.{base64url(hmac_sha256)[:24]}
```

Payload 例：`{"email":"user@lab.edu","tier":"paid","exp":1785600000,"iat":1753123200}`

| 维度 | HMAC（选用 ✅） | Ed25519 公钥签名 ❌ | 在线 license server ❌ |
|------|----------------|---------------------|------------------------|
| 新依赖 | 0（`hmac`/`hashlib`/`base64`/`json` 全是标准库）| 需要 `cryptography` 包 | 需要 VPS + 公网域名 + 证书 |
| 验证延迟 | 离线、瞬时 | 离线、瞬时 | 联网请求，首次激活后还得续期 |
| 密钥泄露后果 | 任何人都能签新 key —— **可接受**（这是发码人方案，不是金融级方案） | 私钥泄露后无法补救 | 被攻破后所有用户都得重发 |
| 复杂度 | 一个 32 字节常量 | KMS / 私钥管理 | 一个完整后端工程 |

桌面版的威胁模型是 **轻量化的"防止顺手扩散"**，不是抵抗有动机的破解者。HMAC 完全够用，且零运维。

签名密钥用 `python -c "import secrets; print(secrets.token_hex(32))"` 生成，硬编码在 `backend/services/license.py` 与 `tools/keygen.py` 同一个常量里。如果将来真的泄露，把常量换掉再发布就行（已激活用户需要重新发码）。

### 2.3 试用计数存储位置

`license.json` 落在 `USER_DATA_DIR/license.json`（即 `%LOCALAPPDATA%\BioSpark-Light\license.json`），与 `biospark.db` 同级。

理由（沿用 v0.1 的"用户数据 vs 程序数据分离"原则）：

- 不在 .exe 旁边 —— Program Files 写权限有限，移动 .exe 不丢状态
- 重装 / 卸载 .exe 时，许可状态保留（用户体验好）
- 多用户操作系统天然隔离

### 2.4 试用计数在训练 _开始_ 时扣减，不在完成时扣

```python
# backend/routers/training.py
ok, msg = license_service.check_can_train()
if not ok:
    raise HTTPException(status_code=402, detail={"message": msg, "paywall": True})
training_manager.start(...)
license_service.record_run()  # 训练已经接受、worker 已 spawn 后立刻 -1
```

不能放在训练完成的 WebSocket 回调里 —— 用户可以中途 Ctrl-C 或关掉浏览器导致计数永不扣减。开始时立刻扣最公平。

### 2.5 前端：全局 fetch 拦截器

```js
const _origFetch = window.fetch;
window._bslFetch = _origFetch;       // 给 license.js 自己用，避免递归
window.fetch = async (...args) => {
    const r = await _origFetch(...args);
    if (r.status === 402) {
        const j = await r.clone().json();
        if (j.detail?.paywall) License.showPaywall(j.detail.message);
    }
    return r;
};
```

业务代码（trainer.js）零侵入，只要后端返回 402 就自动弹付款墙。`License` 自己请求 `/api/license/*` 时用保存的 `_bslFetch`，不会被自己拦截。

## 4. UX 状态机

| 阶段 | 横幅 | 训练按钮 |
|------|------|----------|
| 首次启动 | 🔓 试用模式 · 还剩 5 次训练 | 可用 |
| 训练 N 次后 (N<5) | 🔓 试用模式 · 还剩 (5-N) 次 | 可用 |
| 第 5 次后 | ⚠️ 试用次数已用完 | 点击 → 付款墙 modal |
| 输入合法 key | ✅ 已授权 · user@lab.edu · 至 2027-05-05 | 可用，无限 |
| key 过期后下次启动 | ⚠️ 激活码已过期 | 点击 → 付款墙 |

付款墙 modal 的核心信息：

> 5 次免费试用已结束。打赏 ≥ ¥10（赞赏码 / 爱发电 / Buy Me Coffee）+ 邮箱即可获得：
>
> ① 当前版本激活码（24h 内人工发送）
> ② 下个版本邀请码（v0.3 发布时）

此处刻意把"用户付得到什么"写明白，避免用户怀疑"打了钱不知道得不得到"。

## 5. 运营流程

发码全程手工，没有自动化（v0.2 范围内不做）：

```
用户在赞赏码 / 爱发电留邮箱
        ↓
开发者收到付款通知
        ↓
cd "D:\Claude code\BioSpark-Light\.claude\worktrees\<branch>"
python -c "import sys; sys.path.insert(0,'tools'); \
           from keygen import generate_key; \
           print(generate_key('user@example.com','paid',365))"
        ↓
复制 BSL2-... 到邮件 / 微信发给用户
        ↓
用户在软件里粘贴激活
```

之所以用 `python -c` 而不是 `python tools/keygen.py`，是因为部分聊天客户端的剪贴板会把 `keygen.py` 自动转成 markdown 链接 `[keygen.py](http://keygen.py)`，导致命令在终端里失败 —— `-c` 一行式没有 `.py` 字面量，能稳定复制粘贴。这不是一个有趣的发现，但确实卡过两个小时。

下一版（v0.3）规划自动化：webhook 监听爱发电付款 → VPS 调用 `keygen` → SMTP 发邮件，全程 < 1 分钟到达。

## 6. UI 文案修正：CSV → CSV/TXT

后端实际同时支持 `.csv` 和 `.txt`（OpenBCI GUI 导出的带 `%` 注释头格式），见 [`backend/services/data_preparator.py`](../backend/services/data_preparator.py) 的 `_read_csv_or_txt()`。但 v0.1 的 UI 只写了 "CSV"，OpenBCI 用户看到会以为不能直接拖入 .txt。

修正范围：

| 位置 | 改动 |
|------|------|
| 数据整理 · Mode A 卡片 | `recording.csv` → `recording.{csv,txt}` |
| 数据整理 · Mode B 卡片 | 标题 `CSV + intervals` → `CSV/TXT + intervals` |
| 数据整理 · Mode C 卡片 | `Any ZIP of CSVs` → `Any ZIP of CSV/TXT files` |
| 训练页 · 上传提示 | `CSV (with label column)` → `CSV/TXT (with label column)` |
| `prep.js` Mode B 错误提示 | `expects a .csv file` → `expects a .csv or .txt file` |
| `trainer.js` 错误提示 | `Use .csv or .zip` → `Use .csv, .txt or .zip` |

数据整理介绍段（"training-ready CSV"）和导出按钮（"Download CSV"、"CM (CSV)"、"t-SNE (CSV)"）保持不变 —— 这些是 _输出_，确实只产出 CSV。

## 8. 已知限制与后续工作

| 项 | 现状 | 计划 |
|----|------|------|
| 发码全手工 | 开发者收到付款 → 手敲一条 `python -c` | v0.3：webhook + VPS + SMTP，全自动发码 |
| key 共享无法防 | 用户 A 把 key 分享给 B，B 也能用 | v0.3+：把机器 ID（CPU + 主板序列号 hash）签进 payload |
| 没有吊销列表 | 一旦发出，过期前都有效 | v0.3+：可选的在线 revocation check（仍允许离线 grace period） |
| 试用计数能重置 | 用户删除 `license.json` 即可获得新 5 次 | 不打算修复 —— 决心绕开的人本来就不是付费目标群体；纵深防御性价比低 |
| GitHub Release 仍然手工 unlisted | 通过表单审批 | v0.3 考虑接 Cloudflare Turnstile 自助注册 |
